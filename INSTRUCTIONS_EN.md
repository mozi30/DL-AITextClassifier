The dataset for the first of the three practical assignment submissions (file subm1.csv) has been made available on the e-learning platform. It contains two columns: ID and Text, and 150 rows of examples to be classified.

Please note that, as per the instructions, you must submit two classification proposals for these examples, in two separate files: one based on a model with your own implementation (submission A) and the other in PyTorch (submission B). Transformers must not be used at this stage.



As indicated, the possible labels are: Human, Anthropic, Google, OpenAI and Meta. You must therefore add a column called Labels to the given file, containing a classification from among these 5 for each example (row), whilst retaining the IDs of the examples and their order.



The file must be named according to the following rule: subm1-g<ng>-<course>-_.csv

where: <ng> – will be replaced by the group number; <course> – will be replaced by the course (MEI, MIA, MECD, MMC); sub – will be A for submission A with your own code and B for the PyTorch submission. 

For example: the file for submission A from group 4 of MEI will be named: subm1-g4-MEI-A.csv

In the group’s GitHub repository, you should create a folder called Subm1. This should contain, at a minimum, the two CSV files from the submission and two notebook files containing code that can be run to generate them. These should have the same names as the original files, but with the .ipynb extension. Please note that these notebooks may load a model and classify the given examples, generating the output file. At this stage, you do not need to disclose in the repository the data used or how the models are trained or optimised. In the final report, you must later include an explanation for each of the submitted models.
